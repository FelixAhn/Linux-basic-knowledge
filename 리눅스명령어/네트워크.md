## ifconfig (InterFace Configration)
```bash
네트워크 인터페이스의 활성화/비활성화 및 설정

- ifconfig : 활성화된 네트워크 인터페이스 정보 출력
- ifconfig [인터페이스명] up : 네트워크 인터페이스 활성화
- ifconfig [인터페이스명] down : 네트워크 인터페이스 비활성화
- ifconfig [인터페이스명] [IP주소] : 인터페이스에 IP 주소 설정
- ifconfig [인터페이스명] mtu [값] : MTU(Maximum Transmission Unit) 크기 설정
```

## ip
```bash
ip관련 정보 조회 및 설정

- ip addr : 네트워크 인터페이스와 IP 주소 정보 출력
- ip link : 네트워크 인터페이스 상태 및 링크 정보 출력
- ip route : 라우팅 테이블 정보 출력
- ip addr add [IP주소] dev [인터페이스명] : 특정 인터페이스에 IP 주소 추가
- ip link set [인터페이스명] up : 네트워크 인터페이스 활성화
- ip link set [인터페이스명] down : 네트워크 인터페이스 비활성화
- ip route add [네트워크/마스크] via [게이트웨이 IP] dev [인터페이스명] : 라우팅 추가
```

## netstat (Network Statistics)
```bash
네트워크 프로토콜 통계와 연결상태를 출력

- netstat -n : 숫자로만 출력 (호스트명이나 포트명 대신 IP와 포트 번호 사용)
- netstat -l : 현재 열려 있는 포트 (수신 대기 상태)를 출력
- netstat -t : TCP 연결 정보 출력
- netstat -u : UDP 연결 정보 출력
- netstat -p : 각 연결에 대한 프로세스 정보 출력
- netstat -a : 모든 연결과 열려 있는 포트를 출력
- netstat -tan : TCP 연결 정보를 숫자로만 출력
```

## ss (Socket Statistics)
```bash
네트워크 소켓의 통계와 연결상태를 출력

- ss -t : TCP 연결 상태 출력
- ss -u : UDP 연결 상태 출력
- ss -l : 수신 대기 중인 소켓 출력
- ss -p : 소켓과 관련된 프로세스 정보 출력
- ss -a : 모든 소켓 정보 출력
- ss -s : 소켓 통계 요약 출력
```

## iptables
```bash
패킷 필터링 도구로 패킷의 출입을 제한하는 방화벽구성이나 NAT(Network Address Translation)구성에 사용

- iptables -nL : 숫자 형식으로 모든 체인의 규칙을 출력
    - `-n` : 호스트명이나 포트명을 해석하지 않고 숫자로 표시
    - `-L` : 현재 설정된 규칙 목록을 출력
- iptables -A [CHAIN] -p [프로토콜] --dport [포트번호] -j [ACTION] : 규칙 추가
    - CHAIN: INPUT, OUTPUT, FORWARD 중 하나
    - ACTION: ACCEPT, DROP, REJECT 등
- iptables -D [CHAIN] [규칙 번호] : 지정된 체인에서 규칙 삭제
- iptables -F : 모든 체인의 규칙 삭제
```

## ufw (Uncomplicated FireWall)
```bash
iptables의 제어를 쉽게 하기 위한 도구

- ufw enable : 방화벽 활성화
- ufw disable : 방화벽 비활성화
- ufw allow [포트번호/서비스명] : 특정 포트나 서비스 허용
- ufw deny [포트번호/서비스명] : 특정 포트나 서비스 차단
- ufw status : 현재 방화벽 상태 및 규칙 목록 출력
- ufw delete allow [포트번호/서비스명] : 허용된 포트나 서비스 규칙 삭제
```

## ping
```bash
ICMP 프로토콜의 응답 확인 도구

- ping -c [숫자] [호스트] : 지정한 횟수만큼 대상 호스트에 핑을 전송
    - 예: ping -c 4 google.com (4번 핑을 보내고 응답 확인)
- ping [호스트] : 대상 호스트에 무한 핑을 전송 (Ctrl + C로 종료)
- ping -i [초] [호스트] : 핑 전송 간격을 초 단위로 지정
- ping -s [패킷 크기] [호스트] : 핑 패킷 크기 지정
```

## wget (World wide web + GET)
```bash
웹서버로부터 컨테츠를 가져오는 도구

- wget [URL] : 지정한 URL의 콘텐츠를 현재 디렉토리로 다운로드
- wget -O [파일명] [URL] : 다운로드한 파일의 이름을 지정
- wget -c [URL] : 중단된 다운로드를 이어받기
- wget -r [URL] : 지정한 URL의 내용을 재귀적으로 다운로드
- wget --limit-rate=[속도] [URL] : 다운로드 속도 제한 (예: 100k)
```

## curl (Client for URLs)
```bash
다양한 프로토콜을 사용하여 데이터를 전송하게 해 주는 도구

- curl [URL] : 지정한 URL의 내용을 출력
- curl -O [URL] : URL에서 파일을 다운로드하여 원래 이름으로 저장
- curl -o [파일명] [URL] : URL에서 다운로드한 파일의 이름을 지정
- curl -L [URL] : 리다이렉션을 따라가며 요청
- curl -k [URL] : SSL 인증서를 검증하지 않음
- curl -s [URL] : 출력 내용을 숨김 (조용한 모드)
- curl -I [URL] : HTTP 응답 헤더만 출력
- curl -X [METHOD] [URL] : HTTP 요청 메서드 지정 (예: GET, POST)
- curl -d "[데이터]" [URL] : POST 요청으로 데이터 전송
```

## route
```bash
네트워크의 경로 정보(라우팅 테이블)의 출력, 변경하는 도구

- route : 현재 라우팅 테이블 출력
- route -n : 숫자 형식으로 라우팅 테이블 출력 (호스트명을 IP로 표시)
- route add [네트워크/마스크] gw [게이트웨이 IP] dev [인터페이스명] : 새 라우팅 경로 추가
    - 예: route add -net 192.168.1.0/24 gw 192.168.1.1 dev eth0
- route del [네트워크/마스크] : 라우팅 경로 삭제
    - 예: route del -net 192.168.1.0/24
```
