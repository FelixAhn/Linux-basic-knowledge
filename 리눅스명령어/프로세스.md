## ps (Process Status)
```bash
시스템에서 실행 중인 프로세스에 대한 정보를 출력

- ps -ef : 모든 프로세스를 자세히 출력 (UID, PID, PPID, C 등 포함)
- ps aux : BSD 스타일로 모든 프로세스를 출력 (메모리와 CPU 사용량 포함)
- ps axf : 트리 구조로 프로세스를 출력
- ps axf www : 트리 구조로 프로세스를 출력하며 명령어 줄의 긴 옵션까지 표시
```

## pstree (Process Status TREE)
```bash
시스템에서 실행 중인 프로세스에 대한 정보를 트리구조로 출력

- pstree : 현재 사용자 소유의 프로세스를 트리 구조로 출력
- pstree -p : 각 프로세스의 PID를 함께 표시
- pstree -u : 프로세스를 소유한 사용자 이름을 표시
- pstree -a : 프로세스의 명령줄 인수도 표시
```

## top
```bash
프로세스 목록을 일정 시간마다 새로고침하여 화면에 출력하는 툴
시스템 전반적인 상황을 모니터링 할 수 있음

- 기본 제공 정보: CPU/메모리 사용량, 프로세스 상태, 실행 중인 사용자
- 주요 키: 
    - `q` : 종료
    - `h` : 도움말
    - `k` : 특정 프로세스 종료 (PID 입력 필요)
    - `P` : CPU 사용량 기준 정렬
    - `M` : 메모리 사용량 기준 정렬
    - `1` : CPU 코어별 사용량 보기
```

## nohup (NO HangUPs)
```bash
쉘 스크립트 파일을 데몬 형태로 실행
표준 출력을 지정한 파일로 리다이렉트

- nohup [명령어] : 세션 종료와 관계없이 명령어 실행
- nohup [명령어] & : 백그라운드에서 실행
- nohup [명령어] > [출력 파일] 2>&1 & : 표준 출력과 표준 에러를 지정한 파일로 리다이렉트하며 백그라운드 실행
예: nohup echo "Hello World" > output.txt &
```

## kill
```bash
지정한 프로세스에 저장한 시그널을 보내 프로세스 종료

- kill -2 : `SIGINT` 시그널을 보내 프로세스를 중단 (Ctrl + C와 유사)
- kill -INT : `SIGINT` 시그널을 보내 인터럽트
- kill -15 : `SIGTERM` 시그널을 보내 정상 종료 요청 (기본 옵션)
- kill -TERM : `SIGTERM` 시그널을 명시적으로 보내기
- kill -9 : `SIGKILL` 시그널을 보내 강제로 프로세스 종료 (종료되지 않는 프로세스에 사용, 좀비 프로세스)
- kill -KILL : `SIGKILL` 시그널을 명시적으로 보내기 
```
